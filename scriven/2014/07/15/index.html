(ESXi 5.5): Using Emulex Corporation OneConnect 10gb NICs on IBM hardware#

Recently I started to upgrade our host estate to ESXi 5.5U1 Rollup 2. It soon became apparent that the 10gb NICs weren't communicating correctly. I performed a standard VMware install (not upgrade) using the generic ISO and the driver was installed correctly, yet I couldn't communicate with the vmkernel IP.#

Checking of the switch showed that the MAC address was being presented to it, but no traffic was coming from the host. The problem was therefore host side.#

On closer inspection via the ESXi CLI I noted that the driver for the Emulex card were set to elxnet and not be2net as was the case prior to the upgrade.#

After much reading of the Emulex migration document I then stumbled back onto the Internet and happened upon this article on the HP forums. The final paragraph mentions disabling the elxnet driver and enabling the legacy be2net.#

esxcli system module set --enabled=false --module=elxnet#

esxcli system module set --enabled=true --module=be2net#

reboot#

Although the post wasn't completely related to my issue (I think), the fix was definitely worth a try.#

After running this on the host, network comms started to work immediately! As to why the elxnet driver doesn't work at this time is still a mystery to me. I even went the extra mile and created an OEM IBM ESXi 5.5 ISO and the issue with the elxnet driver persisted.#
